# FlashMind Comprehensive Code Review Report

## Executive Summary

This review covers the entire FlashMind flashcard application codebase, identifying security vulnerabilities, logic bugs, UX issues, performance problems, code quality concerns, and accessibility issues. The application is a FastAPI-based backend with a single-page HTML frontend that uses AI to generate flashcards from PDFs.

**Overall Risk Level: MEDIUM** - Several security and performance issues require immediate attention.

---

## 1. Security Vulnerabilities

### Critical Issues

#### 1.1 Weak Development Secret Key
- **File**: `app/auth.py:20`
- **Severity**: Critical
- **Issue**: Development uses hardcoded insecure secret key `"dev-only-insecure-change-me"`
- **Impact**: Session tokens can be easily forged, allowing complete account takeover
- **Fix**: 
```python
return "dev-only-insecure-change-me"  # REMOVE THIS LINE
raise RuntimeError("SECRET_KEY must be set in development")  # ADD THIS
```

#### 1.2 Missing Production Requirements File
- **File**: `render.yaml:5`, `nixpacks.toml:2`
- **Severity**: Critical
- **Issue**: References `requirements-prod.txt` which doesn't exist
- **Impact**: Deployment will fail, production security dependencies missing
- **Fix**: Create `requirements-prod.txt` with pinned versions and security updates

### High Issues

#### 1.3 Insecure CORS Configuration
- **File**: `app/main.py:35-42`
- **Severity**: High
- **Issue**: Allows all methods and headers with wildcard, credentials enabled
- **Impact**: Potential CSRF attacks, unauthorized API access
- **Fix**:
```python
allow_methods=["GET", "POST", "PUT", "DELETE"],  # Specific methods
allow_headers=["Content-Type", "Authorization"],  # Specific headers
```

#### 1.4 Rate Limiting Bypass
- **File**: `app/api/decks.py:30-38`
- **Severity**: High
- **Issue**: In-memory rate limiting can be bypassed by restarting server or using multiple IPs
- **Impact**: DoS attacks, resource exhaustion
- **Fix**: Implement Redis-based rate limiting with user identification

#### 1.5 File Upload Security Issues
- **File**: `app/api/decks.py:86-98`
- **Severity**: High
- **Issue**: No MIME type validation, file stored temporarily with predictable name
- **Impact**: Malicious file upload, path traversal attacks
- **Fix**:
```python
import magic
mime = magic.Magic(mime=True)
if mime.from_file(file_path) != "application/pdf":
    raise HTTPException(status_code=400, detail="Invalid file type")
```

### Medium Issues

#### 1.6 Insufficient Password Complexity
- **File**: `app/api/auth_routes.py:66-67`
- **Severity**: Medium
- **Issue**: Only 8-character minimum, no complexity requirements
- **Impact**: Weak passwords vulnerable to brute force
- **Fix**: Add complexity requirements (uppercase, numbers, special chars)

#### 1.7 Session Token No Expiration on Logout
- **File**: `app/api/auth_routes.py:118`
- **Severity**: Medium
- **Issue**: Logout only deletes cookie, doesn't invalidate server-side token
- **Impact**: Stolen tokens remain valid until expiration
- **Fix**: Implement token blacklist or shorter expiration times

## 2. Logic Bugs

### High Issues

#### 2.1 Race Condition in Deck Creation
- **File**: `app/api/decks.py:89-95`
- **Severity**: High
- **Issue**: File saved before size validation, temporary file cleanup on error
- **Impact**: Disk space exhaustion, orphaned files
- **Fix**: Validate file size before saving, use proper cleanup with try/finally

#### 2.2 Database Index Missing for Critical Queries
- **File**: `app/main.py:22-24`
- **Severity**: High
- **Issue**: Missing index on study_sessions.date for history queries
- **Impact**: Slow history loading, performance degradation
- **Fix**:
```python
db.study_sessions.create_index([("user_id", 1), ("date", 1)])
db.study_sessions.create_index("date")  # ADD THIS
```

### Medium Issues

#### 2.3 Vector Store Error Handling
- **File**: `app/core/vector_store.py:70-73`
- **Severity**: Medium
- **Issue**: Silent failure in collection deletion
- **Impact**: Orphaned vector data, storage leaks
- **Fix**: Log errors properly, implement retry logic

#### 2.4 Flashcard Review Index Mismatch
- **File**: `static/index.html:1122-1123`
- **Severity**: Medium
- **Issue**: Fallback to array index if card lookup fails, may update wrong card
- **Impact**: Incorrect spaced repetition data
- **Fix**: Ensure proper card indexing or return error if lookup fails

## 4. Performance Problems

### High Issues

#### 4.1 N+1 Query in Deck Loading
- **File**: `app/api/decks.py:146-155`
- **Severity**: High
- **Issue**: Due count calculated per deck in Python instead of database
- **Impact**: Slow loading with many decks
- **Fix**: Use MongoDB aggregation pipeline to calculate due counts

#### 4.2 Inefficient Vector Store Initialization
- **File**: `app/core/vector_store.py:11-24`
- **Severity**: High
- **Issue**: Model loaded on every function call, not cached properly
- **Impact**: Slow response times, high memory usage
- **Fix**: Implement proper singleton pattern with thread safety

### Medium Issues

#### 4.3 Large CSS Bundle
- **File**: `static/index.html:7-435`
- **Severity**: Medium
- **Issue**: All CSS loaded inline, not minified
- **Impact**: Slower initial page load
- **Fix**: Split CSS into separate file, implement minification

#### 4.4 Redundant API Calls
- **File**: `static/index.html:1024`
- **Severity**: Medium
- **Issue**: Stats and history loaded separately on every stats view
- **Impact**: Unnecessary network requests
- **Fix**: Combine into single endpoint or implement caching
